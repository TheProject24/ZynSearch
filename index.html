Engine

Built like a product, not a prototype.

ZynSearch is deliberately split into clear layers so the core remains embeddable while the server and SDKs stay thin, secure, and easy to reason about.

FAST

Fast indexing and retrieval

Documents move through analysis and into the inverted index with a direct, cache-friendly flow.

LOCK

Secure by default posture

HTTP auth plus TLS gives SDK users a straightforward, standard connection pattern.

FLOW

Clear transport boundaries

REST for easy consumption, gRPC for typed integrations, core for embedding.

Pipeline

A search flow you can actually picture.

From query text to ranked results, the system keeps each step visible and understandable. That makes it easier to ship, debug, and extend.

01
Analyze

Normalize incoming content into a stable token stream with the core analyzer.

02
Index

Store document terms in the inverted index and preserve document identity for later lookup.

03
Score

Rank query matches so the best results rise quickly and consistently.

04
Serve

Deliver the same capability through HTTP, gRPC, or embedding without duplicating the engine.

API
